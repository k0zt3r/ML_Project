# LLM-сервис с RAG по документам ИБ

Проект реализует backend-сервис на FastAPI для ответов на вопросы по документам в области информационной безопасности.

## Предметная область

Предметная область: нормативные документы и требования по информационной безопасности.

В корпус входят документы:

- требования ФСТЭК России;
- приказ ФСТЭК России N 21;
- ФЗ-149 "Об информации, информационных технологиях и о защите информации";
- ФЗ-152 "О персональных данных";
- ФЗ-98 "О коммерческой тайне".

Документы лежат в папке `documents/`.

## Архитектура

RAG-пайплайн:

1. Сервис загружает документы из `documents/`.
2. PDF читаются через `pypdf`, ODT через `odfpy`, HTML через `BeautifulSoup`.
3. Тексты разбиваются на чанки с перекрытием.
4. Для чанков строятся эмбеддинги через `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`.
5. Эмбеддинги хранятся в памяти.
6. Для вопроса пользователя считается эмбеддинг и выбираются top-k похожих чанков.
7. В промпт LLM добавляются вопрос и найденные источники.
8. Ответ возвращается вместе с источниками.

## Установка

```bash
pip install -r requirements.txt
```

## Настройка LLM

По умолчанию используется OpenRouter:

```bash
export OPENROUTER_API_KEY="your_api_key"
export LLM_BASE_URL="https://openrouter.ai/api/v1"
export LLM_MODEL="z-ai/glm-4.5-air:free"
```

Можно использовать другой OpenAI-compatible endpoint:

```bash
export OPENAI_API_KEY="your_api_key"
export LLM_BASE_URL="https://api.openai.com/v1"
export LLM_MODEL="gpt-4o-mini"
```

Ключи не сохраняются в коде.

## Запуск

```bash
uvicorn main:app --reload
```

После запуска:

- Swagger UI: `http://127.0.0.1:8000/docs`
- health check: `http://127.0.0.1:8000/health`

## Формат запроса

`POST /ask`

```json
{
  "question": "Какие меры защиты персональных данных должен применять оператор?"
}
```

## Формат ответа

```json
{
  "answer": "Текст ответа модели...",
  "sources": [
    {
      "doc_id": "Федеральный закон от 27 июля 2006 г N 152 ФЗ О персональных данных с изменениями",
      "score": 0.78,
      "snippet": "Фрагмент документа..."
    }
  ]
}
```

## Примеры тестовых вопросов

Информационные вопросы:

- Что относится к персональным данным?
- Какие обязанности есть у оператора персональных данных?
- Что такое коммерческая тайна?
- Какие меры защиты информации описаны в документах?

Уточняющие вопросы:

- Какие требования предъявляются к защите персональных данных в ФЗ-152?
- Что говорится о доступе к информации в ФЗ-149?
- Какие организационные меры защиты упоминаются в приказе ФСТЭК N 21?

Проверка недостатка информации:

- Какие требования предъявляются к возврату товара в интернет-магазине?

Ожидаемое поведение: модель должна ответить, что информации в документах недостаточно.

## Пример curl

```bash
curl -X POST "http://127.0.0.1:8000/ask" \
  -H "Content-Type: application/json" \
  -d '{"question": "Что такое персональные данные?"}'
```

## Ограничения

- Индекс хранится только в памяти и пересобирается при запуске.
- Качество ответа зависит от качества распознавания текста в PDF/ODT.
- Для генерации ответа нужен API-ключ LLM.
- Сейчас нет авторизации пользователей и истории диалогов.

## Возможные улучшения
